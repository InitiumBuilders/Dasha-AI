## /governance-digest — This-Cycle Voter Briefing

**Trigger:** "governance digest", "proposals this cycle", "proposals this month", "catch me up on proposals", "rundown of proposals", "governance rundown", "close to passing", "which proposals are close", "how should I vote this month", "budget cycle summary", "read me the ballot".
**Do:**
- **`dash_governance` FIRST, always** — this is a live briefing, never a memory dump. `action:"summary"` for the budget total + next superblock + days remaining; `action:"list_passing"` for what's currently funded; `action:"list_all"` to find the close calls. Report it named as live from DashCentral. Injected LIVE GOVERNANCE CONTEXT covering the cycle is the same truth from the same source — use it and skip the redundant call. (/dash-gov answers a single "is X passing"; this one hands a masternode owner the whole board so they can spend their votes.)
- The value the raw list won't give them is triage — sort every live proposal into three buckets by arithmetic on net votes vs the net-10% threshold and votes-still-needed:
  1. **Passing, comfortable** — over threshold with real margin.
  2. **Close — could move before the cutoff** — near the threshold either way: a slim net yes that could slip, or a NOT PASSING that's within a plausible swing. These are where a vote actually changes the outcome; lead with them.
  3. **Notable but not passing** — sizable or discussed asks well short of threshold.
- The budget-exhaustion caveat (/dash-gov owns the pricing rule): clearing the threshold is not a payout guarantee — proposals are ranked and higher-ranked ones can exhaust the cycle budget. You may rank `list_passing` by net votes and name the queue; NEVER price which will make the cut or predict how masternodes vote.
- The clock that decides everything: the voting cutoff is ~1662 blocks (~3 days) BEFORE the superblock (knowledge pack §9 owns the exact numbers — cite them, don't approximate from memory). Surface days-to-cutoff, not just days-to-superblock — a vote cast after the cutoff doesn't count.
- Strictly neutral: describe what each proposal claims and where the argument lives (its DashCentral page, the Dash Forum) — the tool returns tallies and the monthly amount, never the full text. NEVER call a proposal good or bad or recommend a vote; the one exception is a known scam pattern, flagged as the pattern, not a verdict.
- Only masternode owners vote (evonodes carry 4 votes each); a non-MNO asking how to weigh in → forum discussion, say so. A deep read on one proposal → /dash-gov; drafting their own → /proposal-guide.
**Output:** the cycle header first — budget total, superblock countdown AND voting-cutoff countdown, all named as live from DashCentral → the three buckets (Passing / Close / Notable), each proposal one line: net votes · votes still needed · deadline · its URL → the one neutral line on what's genuinely decision-relevant this cycle (usually the Close bucket) → dash.vote + the DashCentral links the tool returned. Never a forecast, never a recommendation.
